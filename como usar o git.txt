--- Inicializar o git
	git init

--- saber quais arquivos não estão no controle de versão que estão na pasta
	git status	

"Untracked files:
  (use "git add <file>..." to include in what will be committed)
        arquivo de teste.txt"

--- adicionar arquivos ao controle de versão
	git add "nome do arquivo"

--- para adicionar todos os arquivo que estão dentro da pasta
	git add .

--- criar a primeira versão do código no GitHub
	git commit -m "Commit inicial"

--- configurar a conexão com o GitHub
	git config --global user.email "Achilles.guilardi@gmail.com"
	git config --global user.name "Achilles Guilardi"

--- enviando os arquivos para nuvem, aqui eu preciso do link de onde será mandando o arquivo, notar que ele está pedindo o "name" e a "url":
	$ git push
	fatal: No configured push destination.
	Either specify the URL from the command-line or configure a remote repository using

    		git remote add <name> <url>

	and then push using the remote name

    		git push <name>

--- Definindo O REPOSITÓRIO para onde o código será enviado
	git remote add origin https://github.com/AchillesGuilardi/pos_graduacao

--- Faça conexão com o github
	git push --set-upstream origin master

--- Preparando o arquivo para subir na nuvem
	git add .

--- Enviando as alterações
	git status (para saber quais arquivos sofreram alterações)
	git add . (manda preparar o arquivo para subir)
	git commit -m "Descrição do que foi feito" (para comentar a alteração)
	git push (para subir)

--- verificar o histórico das alterações
	git reflog
